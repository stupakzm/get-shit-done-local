  <action>
    Create get-shit-done/workflows/set-model.md following the GSD workflow file structure (role declaration, step blocks). Study settings.md for the AskUserQuestion and Bash tool patterns — mirror that structure.

    The workflow has these steps in order:

    **Step 1: load_context**
    Run two commands:
    - `node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" ollama list --raw` to get installed models with sizes (handle gracefully if Ollama unreachable — note "Ollama unavailable" in table, still show cloud options in picker)
    - `cat .planning/config.json` (or default to empty config if file absent)

    Parse: installed models array, `config.model_overrides` (or {}), `config.model_profile` (or 'balanced').

    **Step 2: show_view_table**
    Display a markdown table with headers: Role | Model | Source

    Rows are built from ROLE_FRIENDLY_NAMES (all 12 roles). For each agentKey:
    - If `model_overrides[agentKey]` exists → Model = that value, Source = `assigned`
    - Else → Model = profile default from MODEL_PROFILES[agentKey][profile], Source = `profile:<profile>`

    Display table before any picker. This is the CMD-03 view — shown automatically at start.

    **Step 3: role_picker_loop**
    This is the main loop. Repeat until user says no:

    3a. AskUserQuestion: "Which role would you like to configure?" with numbered list of all 12 friendly names + "0. Exit without changes"

    If user picks 0 / Exit → break loop.

    3b. AskUserQuestion: "Select model for [Friendly Name]:" with numbered list:
    - Lines 1..N: `N. <model-name> (<size>)` for each Ollama model from ollama list
    - After Ollama models: fixed cloud options numbered sequentially: `sonnet`, `haiku`, `inherit`
    - Option 0: `0. Reset to profile default`

    Use the Ollama model name exactly as returned by `ollama list` — never accept free-form input. Prepend `ollama:` prefix when writing to config.

    3c. Determine the value to write:
    - If user picked Ollama model → value = `ollama:<model-name>`
    - If user picked cloud option → value = that plain ID (e.g. `sonnet`)
    - If user picked 0 (Reset) → value = `reset`

    3d. Run: `node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" config-set-model-override <agentKey> <value> --raw`

    Parse JSON output. Capture `previousValue` (or 'none' if undefined).

    3e. Display diff: `<FriendlyName>: <previousValue ?? 'profile default'> → <new-value>`
    For reset: `<FriendlyName>: <previousValue> → profile default`

    3f. AskUserQuestion: "Assign another role?" options: ["Yes", "No"]

    If Yes → loop back to 3a.
    If No → break.

    **Step 4: done**
    Display: "Model assignments updated. Run /gsd:set-model again to view the current table."
    If no changes were made (user exited immediately): Display: "No changes made."

    **Discretion decisions (per CONTEXT.md):**
    - View-only mode: no-args default shows the table and exits immediately (skip role picker). Detect by checking if the workflow was invoked with a `--view` flag or simply no positional args — use no-args as the view-only trigger.
    - Cloud models always shown in picker even when Ollama is unavailable.
    - ROLE_FRIENDLY_NAMES mapping: use the 12 roles listed in the interfaces block above.

    **Step 5: update_settings_md**
    Open get-shit-done/workflows/settings.md. Find the `Quick commands:` block in the `confirm` step (currently ends with `- /gsd:plan-phase --skip-verify — skip plan check`). Add the following line immediately after the `/gsd:set-profile` line:

    ```
    - /gsd:set-model — assign a model to a specific GSD role
    ```

    The resulting Quick commands block should read:
    ```
    Quick commands:
    - /gsd:set-profile <profile> — switch model profile
    - /gsd:set-model — assign a model to a specific GSD role
    - /gsd:plan-phase --research — force research
    - /gsd:plan-phase --skip-research — skip research
    - /gsd:plan-phase --skip-verify — skip plan check
    ```

    Do not modify any other part of settings.md.
  </action>
